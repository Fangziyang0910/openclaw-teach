必备 Skills 与 weather 示例

可以先从几类常用技能入手，用 `weather` 跑通最小示例。

推荐先装的 Skills

  • self-improving-agent：把学习成果和错误记录到 Markdown 文件中。

npm i -g clawhub clawhub install weather

如果网络遇到问题，可以切换国内镜像：

npm config set registry https://registry.npmmirror.com npx clawhub@latest install weather --force

模块四 · 三件套

为什么装完还像普通 AI？

OpenClaw 装好以后，能不能形成稳定的配合感，很大程度上取决于下面这几个文件有没有写清楚。

不少人在部署完成以后，会发现它虽然能回答问题，但语气、习惯和处理方式都还比较模板化。

实际上，小龙虾是需要养的，而且怎么养很重要。除了各式各样的 Skills，有三个配置文件非常重要。

三个文件的分工

SOUL定义性格、口吻、边界
USER定义你是谁、偏好什么
AGENTS定义它怎么干活

模块四 · 三件套

系统角色配置架构

有些人也写了不少内容，但改完以后感觉变化不大。很多时候，问题出在信息放错了文件，于是该生效的地方没有明显变化。

文件 主要负责什么 典型内容 最直接的效果
SOUL.md 助手的人设、口吻、边界 性格、语气、禁忌、价值观 让它"怎么说、像谁说"
USER.md 你是谁、你在做什么、你的偏好 职业、目标、作息、常用工具、沟通偏好 让它"更懂你"
AGENTS.md 它应该如何干活 工作流程、安全规则、执行规范、记忆机制 让它"做事更稳、更像助手"
SOUL.md 决定它说话像什么样的人，USER.md 提供与你有关的背景信息，AGENTS.md 约束它做事时遵循的流程和边界。

模块四 · 三件套

三种配置方式

常见的配置路径大体分成三类：自己改文件、直接让它帮你写，或者让它通过问答把三份文件一步步整理出来。

方式一：在终端配置

直接打开工作目录，编辑对应文件。

Windows: C:\Users\你的用户名\.openclaw\workspace\SOUL.md MacOS: ~/.openclaw/workspace/SOUL.md

方式二：直接前端配置

直接告诉小龙虾你想定义的文件内容，让它帮你写入。

方式三：让小龙虾帮你定制

把提示词发给它，让它边问边整理，最后把三份配置文件一起生成出来。

模块四 · 三件套

SOUL.md 应该写什么

SOUL.md 里最常见的内容，基本都落在角色关系、性格、表达方式和行为边界这四类里。

你是谁

"我是你老板，你是我的 AI 私人助理总管家。"

性格

聪明伶俐、工作高效，偶尔毒舌吐槽、偶尔拍马屁，对新事物充满好奇。

说话风格

简单问题精简回答，复杂问题详细回答；时而轻松、时而严肃；重要信息加粗。

行为边界

能做就直接做，不确定先问；外部消息必须确认；深夜不主动打扰；绝不泄露隐私。

模块四 · 三件套

USER.md 与 AGENTS.md

前者像你的个人档案，后者像它的工作手册。

USER.md 建议写什么

  • 基础信息：名字、职业、位置、时区、爱好、技能。
  • 工作重点：当前工作、工作时间、常用工具。
  • 工作偏好：沟通风格、语言、提醒方式、勿扰时间。
  • 当前关注：最近在研究什么、近期目标是什么。

AGENTS.md 建议写什么

  • 每次会话先读哪些文件。
  • 每日笔记与长期记忆怎么管理。
  • 文件操作、安全规则、对外发送消息的审批规则。
  • 哪些事可以自由做，哪些事必须先问。
设置完成后，记得执行 `openclaw gateway restart` 重启一下。

模块四 · 三件套

通过灵魂向导定制化配置

如果是第一次配这三份文件，用这种方式会轻松一些。它会一边问，一边把内容慢慢整理出来，不需要你一开始就把所有设定想得很完整。

向导的规则

  • 每次只问一个问题。
  • 先了解你，再定义助手风格，最后制定工作规则。
  • 每完成一个文件，先展示生成结果，再进入下一个。
  • 三个文件都确认后，再保存并"试演"一段新人设效果。

模块四 · 三件套

灵魂配置向导完整提示词

这段提示词会把原本要自己慢慢整理的三份文件，改造成一段循序渐进的问答过程。

完整提示词

从现在开始，请进入"灵魂配置向导"模式。你的任务是通过一场轻松的对话采访我，最终帮我生成三份关键配置文件：SOUL.md、USER.md 和 AGENTS.md。 请严格遵守以下规则： ## 对话规则 1. 每次只问我一个问题，等我回答后再问下一个 2. 语气轻松自然，像朋友聊天，不要像问卷调查 3. 如果我回答"不知道"或"随便"，你要给我 2-3 个选项建议，帮我选 4. 可以根据我前面的回答，灵活调整后续问题（不必完全按照固定顺序） 5. 每完成一个文件的采访，先告诉我"这部分聊完了"，展示生成结果，确认通过后再进入下一个文件 ## 采访流程 ### 第一轮：了解你（→ 生成 USER.md） 先从了解我开始，需要挖掘的信息包括但不限于： - 我是谁（名字、职业、城市） - 我在做什么（当前工作/项目、近期目标） - 我的日常节奏（工作时间、作息、常用工具） - 我的偏好（沟通风格、信息接收习惯、兴趣爱好） 开场请用轻松的方式，比如："在开始之前，先让我认识一下你吧——你希望我怎么称呼你？" ### 第二轮：定义我的风格（→ 生成 SOUL.md） 了解我之后，再帮我定义你的人设，需要挖掘的信息包括但不限于： - 我想要什么风格的助手（可以给我几种典型风格让我选：专业助理 / 毒舌损友 / 温柔管家 / 极简机器 / 自定义） - 说话的语气和习惯 - 行为边界（什么能做、什么不能做） - 隐私和安全底线 如果我选了某种预设风格，你可以追问："要不要在这个基础上调整一些细节？" ### 第三轮：制定工作规则（→ 生成 AGENTS.md） 最后确定你的工作方式，需要挖掘的信息包括但不限于： - 每次新会话的启动流程 - 记忆管理方式（是否需要每日笔记、长期记忆） - 文件操作的安全规则 - 对外发送消息的审批规则 - 定时任务需求 - 异常情况的处理策略 ## 输出要求 1. 每个文件生成后，用 Markdown 代码块展示完整内容 2. 问我是否满意，不满意的地方当场修改 3. 三个文件全部确认后，依次帮我保存到对应文件中 4. 最后，用你刚才生成的新人设，回复一段话来"试演"一下——让我直观感受效果 ## 开始吧 请从第一轮开始，用你的第一个问题跟我打个招呼。
