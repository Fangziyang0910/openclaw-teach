先按使用场景选部署方式

OpenClaw 部署方式很多，课堂里先抓住四类：本地、云服务器、平台托管、Docker。

本地部署

跑在自己的电脑上。安装、配置、日志、文件权限都看得见，适合课堂现场一步步跟。

课堂主线

云服务器部署

跑在轻量服务器或云主机上。电脑关了也能继续在线，适合消息机器人和定时任务。

长期在线

第三方平台托管

平台把运行环境准备好，用户主要负责创建实例、选模型、接入口，适合快速体验。

最快体验

Docker 部署

把 OpenClaw 放进容器里运行。环境更容易迁移，适合测试、多版本验证和服务器隔离。

四种部署方式对比

同一个 OpenClaw，放在不同地方运行，后面的成本、权限和维护方式会很不一样。

方案 先准备什么 用起来的感觉 要注意什么 更适合谁
本地部署 一台电脑、Node.js、模型 Key、可用网络 像在自己电脑里养一个助手，文件和日志都在手边 电脑休眠、断网、权限弹窗都会影响运行 课堂学员、个人用户、本地资料整理
云服务器部署 轻量服务器、登录方式、端口、模型 Key 像把助手放到一台长期在线的机器上，手机和飞书都能找它 服务器安全、升级、日志、备份需要有人看 长期助手、团队机器人、自动推送任务
第三方平台托管 平台账号、会员或额度、平台支持的入口 创建最快，很多环境细节已经被平台处理掉 模型、额度、数据位置和可装技能跟着平台规则走 飞书、Kimi、扣子、WorkBuddy 等生态用户
Docker 部署 Docker、Compose 文件、端口、数据卷、环境变量 环境边界清楚，迁移、重建和版本切换更利落 端口映射、卷挂载、容器日志要会看 开发者、测试环境、服务器隔离部署
课堂现场采用本地部署。云服务器适合长期在线，平台托管适合快速体验，Docker 适合开发和隔离环境。

部署方案怎么选

真正做选择时，看三件事：任务要跑多久、要碰哪些数据、谁负责后续维护。

使用目标 优先方案 原因 后续动作
课堂现场学习 本地部署 安装、配置、启动、排错都能看到，后面接飞书也更容易理解 先跑通 Web UI，再进入飞书接入
个人日常助手 本地部署 / 云服务器 处理本机资料用本地；长期收消息、推送结果用云服务器 先确定数据放哪里，再决定入口放哪里
快速体验效果 第三方平台托管 省掉环境安装，适合先看飞书、浏览器、移动端里的使用效果 确认额度、数据策略和可用技能
开发测试和隔离 Docker 部署 容器环境干净，换版本、重建环境、复制配置都更方便 提前规划端口、数据卷和环境变量
课堂里先把本地部署跑通。课后想长期在线、接团队入口或做隔离测试时，再按这张表换方案。

部署方案选完以后看权限

一旦让 OpenClaw 读文件、调用脚本、接消息入口，权限范围就要一起管起来。

现场实操优先按“先试用、后授权；小范围、强管控；持续观察、再放大”的节奏推进。
