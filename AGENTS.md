# 项目协作说明：OpenClaw 从入门到案例实践

## 本项目核心任务

本项目用于准备一门线下/现场教学课程，课程名称为：**OpenClaw 从入门到案例实践**。

课程时长为一个上午，目标是让学员在有限时间内完成从认识 OpenClaw 到真实上手部署、接入飞书、配置关键能力并完成案例实战的完整体验。

本项目是一个课程资料与教学项目。后续所有修改都应优先服务于课程交付质量，包括讲知识讲解、案例分析，实操演练等。

## 课程内容

课程包含如下内容：

1. 认识 OpenClaw 是什么，理解它能解决哪些真实问题。
2. 理解 OpenClaw 的基本原理、核心架构和运行逻辑。
3. 了解 OpenClaw 的多种部署思路
4. 在课堂现场手把手带学员在自己的电脑上部署 OpenClaw。
5. 带学员将 OpenClaw 接入飞书，让 OpenClaw 成为可日常使用的助手入口。
6. 带学员完成 Skills 配置，让 OpenClaw 具备可扩展能力。
7. 结合三个案例完成实战演练，让学员理解 OpenClaw 不是玩具，而是可以落地到工作流中的 Agent 工具。

## 大纲与讲稿写作偏好

后续撰写或修改课程大纲、讲师讲稿、PPT 备注时，必须遵守以下偏好：

### 1. 大纲写作偏好

- 大纲只写课程结构、章节层级、讲述要点和可展开内容，不要写给用户的解释、写作理由或协作对话。
- 不要在大纲中出现“这样安排的好处是”“先让学员进入场景”“课堂表达重点”“可以讲成”等元说明。
- 大纲应像正式课程提纲，而不是 AI 对用户的建议稿。
- 大纲标题要自然、简洁，避免生硬概念包装。优先使用“从聊天窗口到真实工作流”这类学员容易理解的表达。
- 大纲可以列出案例、场景、数据、社区现象和发展时间线，但不要使用宣传腔或强行制造热度。

### 2. 讲稿写作偏好

- 讲稿必须是讲师可以直接念给学员听的正文，不要包含“下面是一版”“适合放进备课稿”“课堂上可以”“这里建议”等提示语。
- 讲稿面向普通大众，不默认学员是技术专业用户。技术词可以保留，但第一次出现时要用口语解释。
- 表达要通俗但不失专业性。例如：
  - “长期监听”应改成“持续关注一个入口里的新消息”。
  - “调用接口”可改成“连接外部系统或服务”。
  - “执行脚本”可改成“运行预先写好的自动化操作”。
  - “Gateway”可以解释为“中转和调度服务”。
  - “Skills”可以解释为“教 Agent 使用某种能力的说明书和配套资源”。
- 讲稿要保留自然课堂口吻，可以使用“我们来看一个例子”“换成更口语的说法”等表达，但不要像 AI 助手在解释自己的写法。
- 讲稿段落要适合口播。不要堆长句，也不要把每一句都拆成零散短段。
- 可以使用适度 Markdown 格式增强可读性，例如加粗关键概念、使用列表整理连续要点、少量使用下划线标出概念；不要过度加粗。

### 3. 去 AI 腔要求

- 避免“不是 X，而是 Y”“不仅……而且……”“一句话告诉你”“最后落到……”这类明显 AI 化表达。
- 少用“核心、关键、本质、赋能、生态、路径、闭环、抓手”等空泛词；如确需使用，应落到具体任务或具体场景。
- 删除协作痕迹和讨好语气，例如“您说得完全正确”“希望这对您有帮助”“如果你需要我还可以”等。
- 不要用夸张宣传语。社区数据、名人评价、GitHub stars、项目历史等内容应如实表达，并在正式授课前核实日期和数字。
- 对外部评价要措辞严谨。例如 Moltbook 相关评价不能直接说成“评价 OpenClaw 本体”。

### 4. 课程叙事偏好

- 第一部分“认识 OpenClaw”适合按以下逻辑展开：大众使用 AI 的痛点 → 对话型 AI 到 Agent → OpenClaw 的出现 → OpenClaw 的组成与能力 → OpenClaw 能做什么 → 爆火与社区关注。
- 讲 OpenClaw 时优先从真实使用场景进入，不要先堆定义。
- 讲 Agent 时重点放在“从回答问题到推进任务”，不要把概念讲玄。
- 讲 OpenClaw 的能力时，多讲飞书、资料整理、本地文件、自动化任务、知识库、团队协作等具体场景。

## 参考教程库说明

`awesome-openclaw-tutorial/` 是本项目的重要参考资料目录。它包含 OpenClaw 从入门、部署、配置、扩展到实战应用的完整教学资料。

该目录当前主要包含：

1. `README.md`：教程总入口，包含教程定位、目录、版本说明和资源导航。
2. `LEARNING-PATH.md`：7 天学习路径，可用于辅助设计课程节奏。
3. `Openclaw史上最简单教程，小白一键部署.md`：多平台部署路线合集，可用于部署思路讲解。
4. `docs/`：主线教程，共覆盖入门、安装、快速上手、文件管理、知识库、日程、自动化、Skills、多平台、API、高级配置和实战案例。
5. `appendix/`：附录工具资料，包括命令速查、Skills 清单、API 服务商对比、常见问题、配置模板、最佳实践、视频教程、飞书检查清单等。
6. `examples/README.md`：示例说明文档，可用于案例设计参考。
7. `jvsclaw部署.md`、`宝塔面版claw.md` 等：补充部署方案文章，可用于讲解不同部署路线的取舍。

该目录中的 `.git`、`.vscode`、脚本、站点配置、搜索索引、发布工具和示例代码等无关内容已经删除。后续不要默认恢复这些无关文件，除非用户明确要求。
