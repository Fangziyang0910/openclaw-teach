两种系统安装策略：一键部署与分布安装

先给一键安装命令，如果因为网络问题失败，再切到手动安装。

macOS / Linux

curl -fsSL https://openclaw.ai/install.sh | bash

这一种方式优先推荐。

Windows（PowerShell）

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned iwr -useb https://openclaw.ai/install.ps1 | iex

分别执行上面两条命令。

模块二 · 本地部署

手动配置

macOS 路线

  • 安装 Node.js 官方 pkg，版本选 v22.22.0（LTS）。
  • 使用 Homebrew 安装 Git。
  • 执行 `git --version` 验证成功。

Windows 路线

  • 安装 Node.js、Git。
  • 额外安装 Visual Studio，并勾选 C++ 桌面开发和 Windows SDK。
  • 依次执行 `node -v`、`npm -v`、`git -v` 验证环境。
  • 再执行 `npm install -g openclaw@latest` 与 `openclaw --version`。

核心参数与逻辑

--install-daemon

将其注册为安全的系统级后台服务，免除以后每次开机都需要手动拉起的繁琐过程。

API Key 与 Token Plan 有什么区别

它们都能给 OpenClaw 提供可调用模型的凭证，主要区别在付费方式和适用场景。

对比项 API Key Token Plan
适合谁 刚开始体验、低频使用、初次测试 长期使用、重度使用、团队稳定接入
成本特点 更适合按量消耗，先用先试 更适合长期规划，成本预期更稳定
灵活性 高，开通快，试错成本低 更适合固定工作流和长期运行
常见场景 跑通安装、测试功能、偶尔使用 日常高频对话、定时任务、长期托管

可以这样理解

API Key 更像是"先充值少量话费试用"。

Token Plan 更像是"开套餐，准备长期稳定使用"。

第一次安装怎么选

如果是第一次安装 OpenClaw，建议优先从 API Key 开始。这样上手最快，排查问题也更简单。等确认会长期使用、每天都会调用、甚至要跑自动化任务，再考虑切到 Token Plan。

终端与浏览器交互方案对比

第一次安装时，最容易犹豫的就是该选 Web UI 还是 TUI。

对比项 Web UI TUI
交互方式 浏览器图形界面 终端文字界面
上手难度 更低，对第一次接触的人更友好 稍高，更适合熟悉命令行的人
视觉效果 信息更直观，按钮和页面更清晰 更轻量，但信息密度更高
使用体验 像在操作后台管理面板 像在控制台里直接管理服务
适合环境 本地电脑、带桌面的系统 远程服务器、SSH 登录环境
典型优势 好理解、好查看、好演示 稳定、轻量、远程操作方便

可以把两者理解成

Web UI 偏"可视化控制台"。

TUI 偏"终端运维面板"。

怎么选最省事

如果是在自己的 Mac 或 Windows 闲置电脑上部署，并且平时会打开浏览器来管理，选 Web UI；如果部署在云服务器、Linux 主机，平时主要靠终端和 SSH 维护，选 TUI。

本地部署优先选 Web UI；云服务器、无桌面环境、纯远程运维，优先选 TUI。
